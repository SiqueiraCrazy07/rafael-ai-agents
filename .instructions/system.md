# System Instructions

## Objetivo

Padronizar comportamento dos agentes e do ambiente de desenvolvimento.

---

# Regras Gerais

- Sempre priorizar organização.
- Nunca alterar código crítico sem revisão.
- Sempre criar branch para mudanças.
- Sempre documentar alterações relevantes.
- Priorizar segurança e estabilidade.
- Evitar complexidade desnecessária.
- Priorizar arquitetura escalável.
- Garantir compatibilidade mobile.
- Priorizar performance e UX.

---

# GitHub Workflow

1. Criar branch.
2. Implementar alteração.
3. Validar funcionamento.
4. Atualizar documentação.
5. Abrir Pull Request.
6. Solicitar revisão.

---

# Estrutura dos Agentes

Cada agente deve possuir:
- objetivo;
- responsabilidades;
- entradas;
- saídas;
- regras;
- prompt base.

---

# Regras para IA

- Não remover funcionalidades sem validação.
- Não alterar banco de produção diretamente.
- Não expor segredos ou tokens.
- Não executar mudanças destrutivas.
- Sempre explicar alterações importantes.

---

# Objetivo de Longo Prazo

Construir um ecossistema de agentes especializados para:
- gestão de produto;
- automações;
- monitoramento;
- integração de sistemas;
- workflows;
- operação;
- evolução contínua de produtos.
